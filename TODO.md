# LinguaPop AI — Руководство по установке и использованию

## 1. Как установить Node.js

1. Откройте сайт [https://nodejs.org](https://nodejs.org)
2. Скачайте **LTS** версию (рекомендуется)
3. Запустите установщик и следуйте инструкциям
4. Проверьте установку в терминале:
   ```bash
   node -v
   npm -v
   ```

## 2. Как открыть проект

1. Распакуйте архив с проектом в любую папку
2. Откройте терминал (Command Prompt, PowerShell, Terminal)
3. Перейдите в папку проекта:
   ```bash
   cd путь/к/папке/проекта
   ```

## 3. Как установить зависимости

В терминале, находясь в папке проекта, выполните:

```bash
npm install
```

Дождитесь окончания установки.

## 4. Как получить Mistral API key

1. Перейдите на [https://console.mistral.ai](https://console.mistral.ai)
2. Зарегистрируйтесь или войдите в аккаунт
3. Перейдите в раздел **API Keys**
4. Нажмите **Create API Key**
5. Скопируйте созданный ключ

## 5. Куда вставить API key в расширении

1. Соберите проект (см. пункт 8)
2. Загрузите расширение в браузер (см. пункт 12)
3. Нажмите на иконку расширения → **Настройки** (⚙)
4. В поле **API ключ Mistral** вставьте скопированный ключ
5. Нажмите **Сохранить**

## 6. Как нажать «Проверить ключ»

В настройках, после ввода ключа, нажмите кнопку **Проверить ключ**.

- Если появится сообщение **Ключ работает** — всё в порядке.
- Если **Неверный API ключ** — проверьте, что ключ скопирован полностью.

## 7. Как выбрать модель

В настройках в выпадающем списке **Модель** выберите:

- **Mistral Small (быстрый)** — быстрый и дешёвый вариант
- **Mistral Medium** — баланс скорости и качества
- **Mistral Large (точный)** — максимальное качество

## 8. Как собрать проект

В терминале выполните:

```bash
npm run build
```

Дождитесь сообщения о завершении сборки.

## 9. Где лежит папка build/dist

После сборки готовое расширение находится в папке `dist/` внутри проекта.

## 10. Как открыть chrome://extensions

1. Откройте браузер Chrome, Edge или Brave
2. В адресной строке введите:
   ```
   chrome://extensions
   ```
3. Нажмите Enter

## 11. Как включить Developer Mode

На странице `chrome://extensions` в правом верхнем углу переключите тумблер **Developer mode** (Режим разработчика) в положение **ON**.

## 12. Как загрузить unpacked extension

1. На странице `chrome://extensions` нажмите кнопку **Load unpacked** (Загрузить распакованное)
2. В открывшемся окне выберите папку `dist` из проекта
3. Расширение появится в списке

## 13. Как пользоваться переводом одного слова

1. Откройте любую веб-страницу
2. Выделите **одно слово** двойным кликом или зажатой мышью
3. Рядом со словом автоматически появится popup с переводом
4. В popup показаны: основной перевод, альтернативы, часть речи, произношение, примеры
5. Popup можно перетаскивать мышью, если он закрывает контент

## 14. Как пользоваться переводом фразы

1. Выделите **несколько слов** на странице
2. Появится маленькая кнопка **Перевести** рядом с выделением
3. Нажмите на кнопку — появится popup с переводом
4. В popup есть кнопки **Озвучить** и **Копировать**

## 15. Как пользоваться popup-переводчиком

1. Нажмите на иконку расширения в панели инструментов
2. Введите текст в поле
3. Выберите языки (Авто → Русский или наоборот)
4. Нажмите **Перевести** (или Ctrl+Enter)
5. Результат можно озвучить или скопировать

## 16. Как пользоваться озвучкой

- В popup перевода слова или фразы нажмите кнопку **Озвучить**
- В toolbar popup нажмите **Озвучить** под результатом
- Браузер произнесёт текст голосом

## 17. Как включить/выключить LLM

1. Откройте настройки расширения
2. Найдите переключатель **Использовать LLM**
3. Включите или выключите его
4. Если LLM выключен, будет использоваться fallback-переводчик

## 18. Как настроить fallback

1. В настройках выключите **Использовать LLM**
2. В поле **Fallback-переводчик** выберите **LibreTranslate**
3. В поле **LibreTranslate endpoint** введите адрес сервера (например, `https://libretranslate.de`)
4. Сохраните настройки

## 19. Как добавить сайт в исключения

1. Откройте настройки расширения
2. В разделе **Исключённые сайты** введите домен (например, `example.com`)
3. Нажмите **Добавить**
4. На этом сайте popup перевода больше не будет появляться

## 20. Как попробовать экспериментальный перевод страницы

1. Откройте страницу, которую хотите перевести
2. Нажмите на иконку расширения
3. Нажмите кнопку **Перевести страницу (экспериментально)**
4. Дождитесь перевода видимого текста
5. Чтобы вернуть оригинал, нажмите **Вернуть оригинал**

## 21. Что делать при ошибке «Укажите API ключ»

1. Откройте настройки расширения
2. Введите API ключ Mistral в соответствующее поле
3. Нажмите **Проверить ключ**
4. Убедитесь, что ключ работает

## 22. Что делать при «invalid API key»

1. Проверьте, что ключ скопирован полностью (без пробелов)
2. Убедитесь, что ключ активен в консоли Mistral
3. Если ключ устарел, создайте новый в консоли Mistral
4. Вставьте новый ключ в настройки

## 23. Что делать при «network error»

1. Проверьте подключение к интернету
2. Убедитесь, что сайт `api.mistral.ai` доступен
3. Попробуйте отключить VPN или прокси
4. Подождите несколько минут и повторите запрос

## 24. Что делать при «rate limit»

1. Подождите несколько минут
2. Проверьте лимиты вашего тарифа Mistral
3. При необходимости рассмотрите переход на платный тариф
4. Или временно переключитесь на fallback-переводчик

## 25. Что делать, если popup не появляется на странице

1. Убедитесь, что сайт не в списке исключений
2. Проверьте, что расширение включено на странице `chrome://extensions`
3. Попробуйте обновить страницу
4. Проверьте консоль разработчика (F12) на наличие ошибок
5. Убедитесь, что вы выделили текст (не просто кликнули)
6. Попробуйте горячую клавишу **Alt+T** после выделения

## 26. Что делать, если озвучка не работает

1. Убедитесь, что в браузере установлены голоса для нужного языка
2. В Chrome перейдите в `chrome://settings/languages`
3. Добавьте русский и английский языки
4. Перезагрузите браузер
5. Попробуйте снова

## 27. Что делать, если перевод страницы сломал верстку

1. Нажмите **Вернуть оригинал** в popup расширения
2. Обновите страницу (F5)
3. Это экспериментальная функция — некоторые сайты могут отображаться некорректно

## 28. Какие есть ограничения у экспериментального перевода страницы

- **Динамические сайты** (SPA, React, Vue) могут перезаписать переведённый текст при обновлении данных
- **Shadow DOM** не переводится
- **IFrame** не переводится
- **Верстка** может сломаться на сложных страницах
- **Большие страницы** могут переводиться медленно и потреблять много токенов API
- Повторный перевод уже переведённого текста предотвращается, но не во всех случаях

## 29. Что можно улучшить дальше

- Добавить кэширование переводов для экономии токенов ✅ (уже реализовано)
- Улучшить перевод страниц для SPA-сайтов
- Добавить поддержку Shadow DOM
- Добавить больше fallback-провайдеров
- Реализовать перевод через контекстное меню с выбором языка
- Добавить горячие клавиши ✅ (уже реализовано: Alt+T, Ctrl+Shift+L)
- Добавить тёмную тему ✅ (уже реализовано)

## 30. Какие настройки всё ещё нужно выполнить вручную

- Получить и ввести **Mistral API ключ**
- При желании настроить **LibreTranslate endpoint** для fallback
- При желании добавить **исключённые сайты**
- Проверить, что в браузере установлены **голоса TTS** для русского и английского
- Настроить **горячие клавиши** в `chrome://extensions/shortcuts` при необходимости

## Горячие клавиши

| Клавиша          | Действие                               |
| ---------------- | -------------------------------------- |
| **Ctrl+Shift+L** | Открыть popup переводчика              |
| **Alt+T**        | Перевести выделенный текст на странице |
| **Ctrl+Enter**   | Перевести текст в popup                |
| **Escape**       | Закрыть popup на странице              |

## Кэширование

Переводы автоматически кэшируются на 24 часа. Это экономит API-токены при повторном переводе одинаковых слов и фраз. Чтобы очистить кэш, откройте настройки и нажмите **Очистить кэш**.
